Проект делится на этапы

Каждый этап - это набор фичей, которые необходимо реализовать работая в команде

Проект будет разрабатываться на GitHub

У каждой команды будет свой репозиторий

Все фичи вливаются через Pull-Request, который должен получить два ОК от других участников команды

Подробнее про работу с git в команде

Работа в команде строится самостоятельно

Для каждого этапа будет две контрольные точки

Часть фичей необходимо сделать к промежуточному дедлайну, часть — к финальному

После дедлайна менторы оценивают то, что получилось и говорят, что нужно доработать
